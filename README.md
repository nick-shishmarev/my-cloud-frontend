# Фронтенд дипломного проекта "Файловое хранилище MY CLOUD"

## 1. Пользовательский интерфейс

### 1.1. Меню приложения.
На всех страницах приложения находится меню, состав пунктов 
которого меняется в зависимости от состояния авторизации
пользователя и его статуса.

**Полный список пунктов меню:**
- ГЛАВНАЯ - основная страница приложения;
- РЕГИСТРАЦИЯ - страница регистрации пользователя;
- ВХОД - страница авторизации зарегистрированного пользователя;
- ХРАНИЛИЩЕ - страница управления файловым хранилищем;
- ПОЛЬЗОВАТЕЛИ - страница административного интерфейса;
- ВЫХОД - страница выхода пользователя из своего аккаунта.

### 1.2. Основная страница приложения.

На главной странице, которая открывается при входе в приложение, находится общая информация о приложении.

### 1.3. Страница регистрации
На этой странице находится форма, позволяющая задать 
данные пользователя: логин, пароль, полное имя, e-mail.

При этом вводимые поля проверяютсяна  соответствие требованиям:
- логин — только латинские буквы и цифры, первый символ — буква, длина от 4 до 20 символов;
- email проверяется  на соответствие формату адресов электронной почты. Для проверки  используется регулярные выражения;
- пароль — не менее 6 символов: как минимум одна заглавная буква, одна строчная буква, одна цифра и один специальный символ. Кроме того пароль требуется ввести 
дважды для избежания опечаток. В этой версии приложения пароль пользователя изменить невозможно.

В случае несоответствия  введенных данных требованиям выводится соответствующее сообщение.

При успешной регистрации сразу происходит авторизация пользователя и переход
на страницу управления файловым хранилищем.

###  1.4. Страница авторизации (логин)

Для авторизации необходимо ввести в форму логин и пароль пользователя. При неуспешной проверке выводится соответствующее. В случае успеха происходит переход на страницу управления файловым хранилищем.

При авторизации пользователя в Заголовке страницы выводятсяего статус и полное имя.

### 1.5. Страница управления файловым хранилищем.

Эта страница доступна для любых пользователей. При этом администратор может управлять хранилищами любых пользователей, включая своё собственное. Обычные пользователи  имеют доступ только к своему хранилищу.

На странице отображается список файлов, загруженных пользователем в хранилище, с основной информацией о них: имя файла, размер, комментарий, дата загрузки, дата последнего скачивания;

В строке каждого файла находятся кнопки управления:
- удаление файла;
- переименования файла и изменения комментария;
- просмотра файла средствами браузера. В случае невозможности просмотра в браузере файл скачивается на компьютер пользователя;
- скачивание файла на компьютер пользователя;
- копирование внешней ссылки для доступа к файлу в буфер.

В титульной части списка файлов выводится полное имя владельца хранилища и кнопка для загрузки новых файлов.

При нажатии на кнопку загрузки открывается форма загрузки, в которой можно выбрать нужный файл и опционально ввести желаемое отображаемое имя файла и комментариий. Если отображаемое имя файла не указано, оно приравнивается к оригинальному имени исходного файла. В базе данных приложения оригинальное имя файла сохраняется неизменным, отображаемое имя можно изменят. В списке файлов указывается отображаемое имя. Скачивается файл также под отображаемым именем.

После выбора файла можно подтвердить загрузку кнопкой "Загрузить" или отменить загрузку нажатием на "Х".

Пщ результатам загрузки выводится соответствующее сообщение и можно загружать другой файл или выйти из режима загрузки нажатием на "Х".

Максимальный размер загружаемого файла установлен 10Мб.

### 1.6. Страница административного интерфейса

Эта страница позволяет администраторам управлять пользователями и их файловыми хранилищами.

Страница доступна только пользователям, имеющим признак "администратор" в списке пользователей.

На странице отображается список пользователей с основной информацией о них: логин, признак администратора, полное имя, e-mail, имя папки пользователя для хранения файлов, количество файлов в хранилище и суммарный объем  файлов пользователя в хранилище.

В строке каждого пользователя находятся кнопки управления:
- переход в хранилище пользователя;
- переключение статуса пользователя;
- удаление пользователя.

Необходимо иметь в виду:
- администратор не может удалить самого себя;
- не может лишить самого себя прав администратора;
- не может удалить пользователя, в хранилище которого есть файлы. Это сделано, чтобы избежать появления "бесхозных" файлов в хранилище. После удаления пользователя от него остаётся только пустая папка. В дальнейшем администратор сервера сможет удалить все пустые папки хранилища.
- администратор не может загрузить файл в чужое хранилище. Все остальные операции с файлами в чужомхранилище ему доступны.

### 1.7. Страница выхода

Выводятся кнопки подтверждения и отмены выхода пользователя из приложения.

При выходе авторизация прекращается и приложение приходит к исходному виду.

## 2. Развертывание фронтенда на сервере

В предлагаемом варианте развёртывания приложения на сервере сборка фронтенда осуществляется на сервере из исходников клонируемых из репозитория. Этот процесс описан в инструкции по развёртыванию бэкенда.
